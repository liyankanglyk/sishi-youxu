## 后端规范

### API响应格式

```python
# 成功
{ "success": True, "data": { ... } }
# 失败
{ "success": False, "error": { "code": "AUTH_INVALID_CREDENTIALS", "message": "账号或密码错误" } }
```

### 错误码

AUTH_ / TASK_ / TAG_ / SYNC_ / ADMIN_ / RATE_LIMITED

### 分层架构

API Route → Service → Repository → Database

### 异常处理

禁止直接raise HTTPException，必须使用core.exceptions.BusinessException/NotFoundException

### 数据库规范

所有表含uuid(CHAR36 PK) + created_at + updated_at

软删除：deleted_at字段标记，Repository基类自动注入WHERE deleted_at IS NULL

外键：逻辑外键(UUID字符串)，不做数据库级外键约束

### Redis规范

Key: sishiyouxu:{功能}:{标识}，必须设置TTL

## 部署

### 环境变量（backend/.env）

DATABASE_URL | REDIS_URL | JWT_SECRET | ACCESS_TOKEN_EXPIRE_MINUTES=60 | REFRESH_TOKEN_EXPIRE_DAYS=7 | SMS_PROVIDER=mock | EMAIL_PROVIDER=mock

### 初始化数据库

```bash
mysql -u root -p < scripts/init_db.sql       # DDL + 预设标签 + 权限矩阵 + 系统配置
mysql -u root -p sishi_youxu < scripts/seed.sql  # 超管 + 演示账号 + 演示任务
```

### Systemd 服务

```ini
ExecStart=/opt/sishi-youxu/backend/venv/bin/python -m uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 4
```

> 生产环境部署详见 `docs/04-工程与部署/服务器部署指南.md`。

### Nginx配置要点

upstream ip_hash（WebSocket sticky session）| /api/ → 反向代理 | /ws/ → WebSocket升级 | 前端静态文件try_files

### CI/CD（计划中，Phase 6 待实现）

GitHub Actions 流水线：push main → lint → test → build

## 快速启动

```bash
# 后端
cd backend && pip install -r requirements.txt
uvicorn src.main:app --reload --port 8000

# 用户端
cd web && npm install && npm run dev

# 管理后台
cd admin && npm install && npm run dev

# 测试
cd backend && python scripts/run_tests.py

# API文档
http://localhost:8000/docs
```
