# 服务器部署指南

> 版本：V1.0 | 更新日期：2026-06-28
>
> 适用场景：将四时有序部署到云服务器，通过域名（不带端口号）访问。

---

## 一、架构概览

```
用户浏览器 (https://example.com)
       │
       ▼
   ┌─────────┐
   │  Nginx  │  ← 端口 80/443，统一入口
   │ 1.24+   │
   └────┬────┘
        │
        ├── /api/*           → 代理 → Backend (127.0.0.1:8000)
        ├── /admin/*         → 静态文件 (admin/dist/)
        └── /*               → 静态文件 (web/dist/)
```

- **Backend**：FastAPI，内部端口 8000，仅 Nginx 直连
- **Web 用户端**：Vue 3 静态文件，`npm run build` 产物
- **Admin 管理端**：Vue 3 静态文件，`npm run build` 产物
- **数据库**：MySQL 8.0+ + Redis 6.0+

## 四、部署步骤

### 4.1 拉取代码

```bash
# 部署目录
sudo mkdir -p /opt/sishi-youxu
sudo chown $USER:$USER /opt/sishi-youxu

cd /opt/sishi-youxu
git clone <你的仓库地址> .
```

### 4.2 后端部署

```bash
cd /opt/sishi-youxu/backend

# 创建虚拟环境
python3.12 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
```

编辑 `.env`，修改以下关键项：

```ini
APP_ENV=prod
DEBUG=false
HOST=127.0.0.1
PORT=8000

# 数据库 — 用上面创建的账号密码
DATABASE_URL=mysql+aiomysql://sishi:<数据库密码>@127.0.0.1:3306/sishi_youxu?charset=utf8mb4

# Redis
REDIS_URL=redis://127.0.0.1:6379/0

# JWT — 务必修改！用 `openssl rand -hex 32` 生成
JWT_SECRET=<随机64位密钥>

# CORS — 填写你的域名
CORS_ORIGINS=["https://example.com"]
CORS_ALLOW_CREDENTIALS=true
```

初始化数据库：

```bash
# 建表 + 预设标签 + 权限 + 系统配置
mysql -u sishi -p sishi_youxu < scripts/init_db.sql

# 超管账号 + 演示数据
mysql -u sishi -p sishi_youxu < scripts/seed.sql
```

验证后端启动：

```bash
source venv/bin/activate
python -m uvicorn src.main:app --host 127.0.0.1 --port 8000

# 另开终端验证
curl http://127.0.0.1:8000/health
# → {"success":true,"data":{"status":"ok",...}}
```

### 4.3 前端构建

```bash
# 用户端
cd /opt/sishi-youxu/web
npm install
npm run build          # 产物在 dist/

# 管理端
cd /opt/sishi-youxu/admin
npm install
npm run build          # 产物在 dist/
```

### 4.4 Nginx 配置

创建配置文件 `/etc/nginx/sites-available/sishi-youxu`：

```nginx
# ── HTTP → HTTPS 重定向 ──
server {
    listen 80;
    server_name example.com;

    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

# ── HTTPS 主站 ──
server {
    listen 443 ssl http2;
    server_name example.com;

    # SSL 证书（见第五节）
    ssl_certificate     /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    # 安全头
    add_header Strict-Transport-Security "max-age=63072000" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "DENY" always;

    # 上传限制
    client_max_body_size 10m;

    # ── API → 后端 ──
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Request-ID      $request_id;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }

    # ── WebSocket → 后端 ──
    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade           $http_upgrade;
        proxy_set_header Connection        "upgrade";
        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }

    # ── 管理端静态文件 ──
    location /admin {
        alias /opt/sishi-youxu/admin/dist;
        try_files $uri $uri/ /admin/index.html;

        # 缓存策略
        location ~* \.(js|css|png|jpg|svg|woff2?)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # ── 用户端静态文件 ──
    location / {
        root /opt/sishi-youxu/web/dist;
        try_files $uri $uri/ /index.html;

        location ~* \.(js|css|png|jpg|svg|woff2?)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/sishi-youxu /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default   # 移除默认站点

# 检查语法
sudo nginx -t

# 重载
sudo systemctl reload nginx
```

### 4.5 Systemd 服务（后端守护进程）

创建 `/etc/systemd/system/sishi-youxu.service`：

```ini
[Unit]
Description=四时有序 Backend
After=network.target mysql.service redis.service
Wants=mysql.service redis.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/sishi-youxu/backend
Environment=PATH=/opt/sishi-youxu/backend/venv/bin
ExecStart=/opt/sishi-youxu/backend/venv/bin/python -m uvicorn src.main:app --host 127.0.0.1 --port 8000 --workers 4
Restart=always
RestartSec=5

# 日志
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable sishi-youxu
sudo systemctl start sishi-youxu

# 查看状态
sudo systemctl status sishi-youxu

# 查看日志
sudo journalctl -u sishi-youxu -f
```

## 九、安全建议

1. **修改默认密码**：首次部署后立即登录 `admin/123456` 并修改密码
2. **JWT 密钥**：生产环境务必用 `openssl rand -hex 32` 生成强随机密钥
3. **数据库密码**：不要使用弱密码，不要暴露在公网
4. **Redis 密码**：生产环境建议 `redis-cli config set requirepass <密码>`
5. **防火墙**：数据库端口 3306、Redis 端口 6379 不要对外开放
6. **日志脱敏**：生产环境设置 `LOG_LEVEL=WARNING`，避免记录敏感信息
7. **定期备份**：建议每日 cron 自动备份数据库

## 十、故障排查

| 现象 | 排查方向 |
|------|---------|
| 502 Bad Gateway | `systemctl status sishi-youxu` 看后端是否运行；检查端口 8000 |
| 静态资源 404 | 检查 `npm run build` 是否成功；Nginx root/alias 路径是否正确 |
| 登录失败 | 检查 MySQL 是否运行；`.env` 中数据库连接是否正确 |
| CORS 报错 | 检查 `.env` 中 `CORS_ORIGINS` 是否包含你的域名 |
| 强制登出不生效 | 检查 Redis 是否运行：`redis-cli ping` |
| nginx -t 报错 | 逐条检查 server_name / root / alias 路径是否存在 |
