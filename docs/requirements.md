## 1. 项目概述

### 1.1 项目背景

**四时有序（Sishi Youxu）** 是一款基于 **艾森豪威尔矩阵**（Eisenhower Matrix）的个人任务管理 Web 应用。用户将任务按「重要性」和「紧急性」两个维度分配到四个象限，从而获得清晰的任务优先级视图。

### 1.2 项目目标

- 提供直观、流畅的四象限拖拽体验
- 支持桌面端和移动端，覆盖日常任务管理全场景
- 纯前端 + 本地存储，无需账号注册，数据完全由用户掌控
- 可作为 PWA 安装到设备主屏幕，支持离线使用

### 1.3 目标用户

- 希望用艾森豪威尔矩阵管理个人任务的用户
- 需要离线可用的任务管理工具的移动端用户
- 偏好简洁工具而非重型项目管理软件的用户

## 4. 数据模型

### 4.1 Task（任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` (UUID) | 主键 |
| `title` | `string` | 任务标题 |
| `posX` | `number` (0-1) | 紧急度水平位置 |
| `posY` | `number` (0-1) | 重要度垂直位置 |
| `urgencyLevel` | `number` (-4~4) | 紧急性等级，由 posX 派生 |
| `importanceLevel` | `number` (-4~4) | 重要性等级，由 posY 派生 |
| `dueDate` | `string \| null` | ISO 日期字符串 |
| `tags` | `string[]` | 标签 ID 数组 |
| `note` | `string` | 备注文本（支持 Markdown） |
| `recurrence` | `null \| 'daily' \| 'weekly' \| 'monthly'` | 重复规则 |
| `generatedNextId` | `string \| undefined` | 自动生成的下一期任务 ID（恢复/撤销时精确清理） |
| `completed` | `boolean` | 完成状态 |
| `completedAt` | `string \| null` | 完成时间 ISO |
| `createdAt` | `string` | 创建时间 ISO |
| `updatedAt` | `string` | 更新时间 ISO |

### 4.2 Tag（标签）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` (UUID) | 主键 |
| `name` | `string` | 标签名称 |
| `color` | `string` (HEX) | 标签颜色 |
| `isPreset` | `boolean` | 是否预设标签 |

### 4.3 象限判定规则

```
posY >= 0.5（重要）
  ├── posX >= 0.5 → Q1（重要 + 紧急）
  └── posX <  0.5 → Q2（重要 + 不紧急）
posY <  0.5（不重要）
  ├── posX >= 0.5 → Q3（不重要 + 紧急）
  └── posX <  0.5 → Q4（不重要 + 不紧急）
```

### 4.4 截止日期状态判定

```
dueDate <  今天 0:00    → overdue  已过期
dueDate === 今天        → today    今天
今天 < dueDate ≤ 周六   → thisWeek 本周
dueDate >  本周         → normal   正常
无 dueDate              → none     无日期
```

### 6.1 跨平台架构

```
┌─────────────────────────────────────┐
│         React 19 + TypeScript       │
│   Zustand · dnd-kit · Dexie · uuid  │  ← 一套代码 100% 复用
├─────────────────────────────────────┤
│           Vite 8 (Rolldown)         │
│     --mode web → PWA Service Worker │  ← Web 构建
│     --mode capacitor → 无 PWA       │  ← 原生构建
├────────────┬────────────┬───────────┤
│  dist/     │  android/  │  ios/     │
│  Web+PWA   │  Capacitor │ Capacitor │
│  ↓ 部署    │  ↓ 打包    │  ↓ 打包   │
│  任意服务器 │  .apk/.aab │  .ipa     │
└────────────┴────────────┴───────────┘
```

- **Web 模式**：`vite build --mode web`，启用 PWA Service Worker，生成可离线缓存的静态站点
- **Capacitor 模式**：`vite build --mode capacitor`，使用相对路径 `./`，不生成 SW（WebView 自带离线能力），构建产物自动同步到 `android/` 和 `ios/` 原生工程
- **所有业务代码**（React 组件、Zustand store、Dexie DB、dnd-kit 拖拽）在三个平台上 **完全相同**，无需任何 `if (platform === 'web')` 条件判断
